Лабораторная работа №1
Цветовые модели: RGB, XYZ, Lab
Вариант: RGB ↔ XYZ ↔ LAB
1. Цель работы

Изучить цветовые модели RGB, XYZ и CIE Lab, реализовать приложения для интерактивного преобразования цвета между ними, исследовать преобразования и особенности цветовых пространств.

2. Постановка задачи

Необходимо разработать веб-приложение, позволяющее пользователю:

выбирать цвет из палитры;

задавать точные значения цветовых координат вручную;

изменять цвет с помощью ползунков;

отображать цвет в трёх моделях одновременно: RGB, XYZ, Lab;

автоматически пересчитывать значения во всех моделях при изменении любой координаты;

выводить предупреждения при выходе цвета за доступный диапазон sRGB.

Приложение должно быть доступно онлайн, а исходный код размещён в репозитории GitHub.

3. Краткое описание используемых цветовых моделей
3.1. RGB

Аддитивная модель, основанная на смешении компонентов Red, Green, Blue.
Используется в мониторах, графике и веб-разработке.

Диапазоны:

R, G, B ∈ [0; 255]

3.2. XYZ (CIE 1931)

Линейная модель, основанная на измерениях чувствительности человеческого глаза.
Используется как промежуточная модель для переводов.

Белая точка: D65
Диапазон значений X, Y, Z зависит от конкретного цвета (обычно до ~95–110).

3.3. Lab (CIELAB)

Перцептуальная модель, приближенная к восприятию человека.

Диапазоны:

L ∈ [0; 100]

a ∈ [-128; 127]

b ∈ [-128; 127]

5. Структура интерфейса

Интерфейс состоит из:

Левой панели — выбор цвета, HEX поле, быстрый выбор модели.

Правой панели — блоки с тремя моделями (RGB/XYZ/Lab), каждая содержит:

ползунок компоненты

числовое поле

Swatch-образец с текущим цветом

Предупреждение при ошибке преобразования

6. Алгоритмы преобразования
RGB → XYZ

Используется формула согласно стандарту sRGB D65, с гамма-коррекцией.

XYZ → Lab и обратно

Используется стандарт CIE 1976 с белой точкой D65.

XYZ → RGB

Выполняется обратное линейное преобразование и обратная гамма-коррекция.

В приложении используются корректные матрицы и функции нелинейности.

7. Проверка корректности работы

Проведён ряд тестов:

изменения любого параметра приводят к корректному пересчёту всех остальных;

переход RGB → XYZ → RGB даёт стабильный обратный цвет (в пределах округления);

при вручную введённых некорректных XYZ значения RGB корректно обрезаются и выдаётся предупреждение;

HEX всегда синхронизирован с RGB.

Все тесты пройдены успешно.
